# OM Outreach Templates — Subtype-Specific Cadences

> **Purpose:** Executable cold outreach copy for each of the five OM subtypes defined in `om-identification-field-guide.md`. Each subtype gets a LinkedIn first-touch DM, a 3-email cold sequence, and a slow-nurture touch for prospects who don't convert in the first cycle.
>
> **How to use:** After classifying a prospect (60-second flowchart in the field guide), pull the matching subtype block below. Personalize the bracketed fields. Send. Log the touch in HubSpot. Move on.
>
> **Editing rules:**
> 1. Never lengthen these. They are tuned to be readable on a phone in under 15 seconds.
> 2. Personalize the `[bracketed]` fields and one specific detail per message. No generic sends.
> 3. Never send the same template twice in 90 days to the same company, even to a different person.
> 4. If a template stops working (reply rate drops below 4% over 30 sends), mutate it and log the change in the changelog at the bottom of this file.

---

## Universal rules across all subtypes

- **Send window:** Tuesday–Thursday, 9:30–11:00 AM Pacific. Avoid Mondays (inbox triage day) and Fridays (mental checkout). Sarah-subtypes A/B/C respond best in the late morning window. Subtypes D/E respond at any hour but engage most thoughtfully Tuesday afternoon.
- **From line:** "Doug @ Culinary Block Collective" — name first, company second, no title. Title-stuffing reads like sales sequence.
- **Signature:** Three lines max. Name. Phone. Website. No headshot, no banner, no "schedule a meeting" button.
- **Tracking:** Use UTM tags on any link (`?utm_source=outbound&utm_medium=email&utm_campaign=om_[subtype]_[touch#]`). This is how you'll learn which subtype converts where.
- **One CTA per message. Always.**

# Reply-Triage Decision Tree

When a prospect replies, classify the reply type before drafting your response. This is what your `cbc-inbound-responder` skill should be doing.

| Reply type | What it looks like | Right next move |
|---|---|---|
| **Hot interest** | "Yes, send the quote" / "When can you do a tasting?" / "What's pricing for X?" | Reply within 60 minutes, even if it's just "got it, sending details by EOD." Send full details same day. |
| **Soft interest** | "Interesting, let me think about it" / "Maybe in Q[X]" | Acknowledge, ask one clarifying question, propose a calendar reminder for the timeline she named. Do NOT push. |
| **Polite decline** | "We're happy with our current vendor" / "Not in market right now" | One graceful response. Drop into 6-week slow nurture. Don't argue. |
| **Operational test** | "What's your COI / minimum / cancellation policy?" | This is a buy signal disguised as a question. Answer all three, even if she only asked one. Same-day response. |
| **Strategic question** | "How do you handle scaling?" / "What's your sustainability story?" | Marcus-or-D-leaning prospect. Respond with substance, not deflection. Offer to send a one-pager. |
| **Silence after the cold sequence** | No reply to all 3 emails | Drop into nurture. The prospect is not dead, she's just not in market today. 70% of B2B closes happen after 5+ touches. |
| **Negative or harsh reply** | "Stop emailing me" / "We're not interested" | One-line acknowledgment ("Got it, removing you — apologies for the noise"), remove from sequence, move on. Never argue. |

# Where to put these files (operator notes)

You have OpenClaw running with five skills. Two of them — `cbc-inbound-responder` and `cbc-linkedin-prospecting` — both need to read this template file. The right pattern is one shared reference file, not duplicated copies inside each skill folder.

### Recommended file structure

```
~/.openclaw/workspace/cbc-skills/
├── AGENTS.md
├── shared/
│   ├── om-personas-deep-dossier.md          ← from earlier conversation
│   ├── om-identification-field-guide.md     ← subtype detection + profile template
│   └── om-outreach-templates.md             ← THIS FILE
├── cbc-inbound-responder/
│   └── SKILL.md
├── cbc-linkedin-prospecting/
│   └── SKILL.md
├── cbc-pipeline-monitor/
│   └── SKILL.md
└── ...
```

### Updates needed in your skill files

**In `cbc-linkedin-prospecting/SKILL.md`,** add a reference at the top:

**In your project root `CLAUDE.md`,** add a section:

```markdown

The /shared/ directory contains the persona and templates layer for all CBC sales-related skills. When working on outreach, prospecting, or inbound response, always consult these in order:

1. om-identification-field-guide.md — classify the prospect
2. om-outreach-templates.md — draft the message
3. om-personas-deep-dossier.md — sanity-check the tone

These files are shared infrastructure. Update them as you learn from real outreach; do not duplicate them inside individual skill folders.
```

### Why this structure

- **Single source of truth.** When you iterate on the templates (you will, every week, based on real reply data), you update one file and every skill picks up the change.
- **Composable across skills.** The `cbc-pipeline-monitor` skill can also read these files when summarizing prospect activity, so its summaries use the same vocabulary as your outbound messages.
- **Survives skill rewrites.** When you eventually rewrite or replace one of the OpenClaw skills, the persona and template layer doesn't get lost — it lives outside the skill folders.
- **Bind-mount safe.** The OpenClaw sandbox bind mount restriction means everything has to live under `~/.openclaw/workspace/`. This structure stays inside that boundary.

### Optional: HubSpot custom fields

To make the subtype actionable in your CRM, add these custom fields to the HubSpot contact object:

| Field name | Type | Values |
|---|---|---|
| `om_subtype` | Single-select dropdown | A · Veteran / B · Promoted EA / C · Accidental / D · Workplace Experience / E · OM/HR Hybrid / Marcus / Unknown |
| `subtype_confidence` | Single-select | High / Medium / Low |
| `last_subtype_review` | Date | Last time you confirmed or updated the classification |
| `current_cadence_step` | Number | Which touch in the sequence (1, 2, 3, 4, nurture) |

After 50–100 prospects, you can run reply-rate-by-subtype reports and learn where to concentrate prospecting effort. That's the data flywheel.
